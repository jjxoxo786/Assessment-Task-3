Task 3: HTML & CSS to create a table displaying information on 10 kinds of web attacks.

Name Web Attacks
SQL Injection
Source: PingIdentity.com
 SQL Injection is a technique where attackers insert malicious SQL queries into user input fields to manipulate the database into divulging sensitive data. This can result in unauthorized access of confidential data such as usernames and passwords. This vulnerability is taken advantage of when user input is not properly sanitized.

More info: W3Schools SQL Injection
Cross-Site Scripting (XSS)
Source: Spanning.com
 XSS allows attackers to inject malicious scripts into trusted webpages visited by other unsuspecting users. The malicious script is executed in the browser when users visit the compromised website which can result in stolen cookies, keystrokes being captured, spread of malware or redirection to harmful websites. XSS occurs when web applications fail to validate or encode user input properly.

More info: OWASP XSS
Cross-Site Request Forgery (CSRF)
Source: Spanning.com
 CSRF tricks an authenticated user into unknowingly executing unwanted actions on a web application. This happens when a user that is logged in to a web application unknowingly opens links or forms sent by the attacker containing a malicious request to the target application. The target application then, executes the malicious request believing it to be legitimate(as the user is logged in), resulting in unauthorised actions such as changing passwords or transferring funds. Strong anti-CSRF tokens can help prevent this type of attack.

More info: CloudFlare CSRF
Brute Force Attack
Source: AppCheck-ng.com
 A brute force attack involves systematically guessing passwords or logging credentials until the correct one is found(trial and error process). Automated tools testing various different credential combinations make this process faster and more effective for attackers. Brute force attacks can be prevented by using strong passwords and account lockout policies.

More info: Fortinet Brute Force Attack
Man-in-the-Middle (MITM)
Source: PandaSecurity.com
 MITM attacks occur when an attacker positions themselves between two unknowing parties. This allows the attacker to secretly intercept and alter communication between the two parties resulting in data theft, credential stealing, and injection of malicious content. Using HTTPS and secure networks can greatly reduce MITM risks.

More info: Fortinet MITM
Denial of Service (DoS)
Source: Cloudflare.com
 A DoS(denial of service) attack aims to make a service unavailable to its legitimate users by overwhelming it with excessive requests. Distributed versions (distributed denial of service) use multiple systems to launch attacks simultaneously, generating more traffic and making the attack more effective. Web application firewalls and traffic filtering can mitigate DoS attacks.

More info: NCSC.gov DoS
Directory Traversal
Source: Fastly.com
 Directory Traversal attacks exploit improper file path handling(ie inputing '../' characters to navigate server's files/directories) to access restricted files. This can expose configuration files, passwords, and other sensitive data. Proper input validation and path sanitization are a few strategies to prevent this attack.

More info: OWASP Path Traversal
Remote File Inclusion (RFI)
Source: Cobalt.io
 Remote File Inclusion(RFI) attack occurs when a web application dynamically includes files based on user input without proper validation. This allows attackers to include and execute malicious remote files on the user's web server. Disabling remote file inclusion in server configuration and sanitizing user input can help prevent RFI.

More info: OWASP RFI
Phishing
Source: Bolddesk.com
 Phishing is a social engineering technique that uses deceptive emails or websites to trick users into revealing sensitive information. These attacks often mimic legitimate companies or impersonate an acquaintance to gain trust. Awareness training and email filtering can help prevent phishing.

More info: Phishing.org
Session Hijacking
Source: Umbc.edu
 Session Hijacking occurs when an attacker takes over a valid user session on a web application allowing the attacker to impersonate the victim and gain unauthorised access to the web server. This can be done by predicting or stealing the user's session ID stored in cookies through XSS, sniffing, or malware. Using secure cookies and HTTPS can help mitigate this risk.

More info: OWASP Session Hijacking
